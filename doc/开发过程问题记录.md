
## 根据TinyRenderer中切换 sandbox 实现逻辑，在 UI 选择Shadow Rendering之后，能够正确渲染出球体和plane，sandbox默认开启键盘交互，但是在控制键盘ASWD键观察渲染的球体和plane的时候，发现球体和plane的相对位置（遮挡关系）会发生变化，这是为何？
WASD 移动相机时，球和地面的遮挡关系“乱跳”，不主要是物体真的在动，而是最终画面上没有按 3D 深度正确遮挡。
### 1. 主要原因：BasePass 没有深度缓冲（引擎问题）
看到的 PBR 光照结果来自 BasePass 的前向绘制，而不是 G-Buffer 里那一遍。
BasePass 虽然开了 enableDepthTest，但 FBO 上没有绑定深度缓冲，深度测试实际上不起作用。
于是物体按 固定绘制顺序 叠在一起：
先画球（GetRenderCommands 里第一个），再画地面
在屏幕重叠区域，后画的地面会盖住球，和相机远近无关。
相机一移动，重叠区域变了，你就会觉得“谁挡谁”在变——这是 画家算法（按提交顺序），不是真实 3D 遮挡。

### 2. 处理
在BasePass::OnInitialize()中新增深度附件,同时做好深度遮罩的设置即可
```c++
void BasePass::OnInitialize()
{
    // ...
    // Setup outputs (forward_depth is used for depth testing only, not sampled downstream)
    mConfig.outputs = {
        {"BaseColor", "basecolor", RenderTargetFormat::RGBA8},
        {"ForwardDepth", "forward_depth", RenderTargetFormat::Depth24}
    };
    ...
}

void BasePass::Execute(const std::vector<RenderCommand>& commands)
{
    //...
    // Apply render settings
        ApplyRenderSettings();

        if (mConfig.clearDepth)
        {
            glClear(GL_DEPTH_BUFFER_BIT);
        }
        glDepthMask(GL_TRUE);

        // Bind input textures
        BindInputs();
    //...
}
```
